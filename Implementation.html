IMPLEMENTACIÓ DEL PROJECTE

Aquest document forma part de la memòria del Treball de Final de Màster de Ciència de Dades de la UOC. Inclou les tècniques utilitzades per generar el conjunt de dades, la Xarxa Neuronal Artificial i els mapes de previsió del risc resultants de l’aplicació del model desenvolupat.

GENERACIÓ DEL DATASET

Generar el conjunt de dades ha estat costós. Com s’explica a la memòria, la majoria d’estudis analitzats relacionats amb la previsió del risc d’incendi i machine learning, analitzen les dades meteorològiques a nivell mensual i utilitzen valors agregats.

Per afegir una novetat a aquest projecte, s’ha aprofitat l’API del Servei Meteorològic per poder iterar, i extreure les condicions meteorològiques específiques de cada dia en el qual es va detectar un incendi. Per fer-ho, s’ha generat un grid sobre de Catalunya de punts separats entre ells per 200m de distància.

Per a cada punt s’ha extret informació de:

  • Altitud
  • Pendent
  • Orientació
  • Coberta del sòl
  • Temperatura
  • Humitat relativa
  • Velocitat del vent
  • Si hi ha foc o no

Cadascuna d’aquestes variables prové d’una capa tipus ràster. L’altitud, pendent orientació provenen d’un Model Digital del Terreny descarregat de la web de l’ICGC. Les cobertes del sòl provenen d’un ràster també descarregat de l’ICGC. Les dades meteorològiques s’han generat a partir d’interpolacions de dades puntuals de les estacions meteorlògiques disponibles a l’API del METEOCAT.

L’API del METEOCAT no és d’ús obert a tothom, però com a estudiant he pogut tenir accés de manera gratuita. El Centre Forestal de Catalunya (CREAF) ha desenvolupat una llibreria que permet rebre dades de prediccions i històrics des d’R.

S’han analitzat un total de 561 incendis històrics. Han representat un total de 561 taules amb 802.591 registres cadascuna (corresponents a la informació de les 8 variables per cadascún dels punts en què s’ha dividit el grid), en total més de 450 milions de registres. Aquesta informació s’ha organitzat en llistes de taules, com que la capacitat de l’equip amb que s’ha dut a terme el projecte és limitada s’ha hagut de dividir aquesta iteració en 7 fragments.

Finalment, s’ha fet un mostreig estratificat que ha permés obtenir un dataset homogeni pel que fa a registres amb foc i registres sense foc.

L’objectiu final és obtenir un model que permeti identificar en quines condicions pot haver-hi foc i en quines no. Per tant, voldrem un model que ens classifiqui bé els casos positius, el casos negatius també, però serem tolerants am els falsos positius, ja que aquests encara que no presentin foc, reuniran les condicions per tenir-ne, i per tant, seran punts potencials per tenir en compte a l’hora de valorar el risc d’incendi.

# Loop per extreure les dades meteorològiques:
for (data in as.list(llistat_dates[data])) {

  # Incendis data:
  foc <- incendisCat %>% 
        filter(DATA_INCEN == data)
  
  
  #Extreu dades Meteo

  dadesMeteo_sf <- dadesMeteo %>% 
    filter(timestamp == data) %>% 
    dplyr::select(max_temperature, mean_relative_humidity, mean_wind_speed) %>% 
    mutate(max_temperature = as.numeric(max_temperature),
           mean_relative_humidity = as.numeric(mean_relative_humidity),
           mean_wind_speed = as.numeric(mean_wind_speed)) 
  
  
  if (nrow(dadesMeteo_sf) == 0) {
    
    n_list = n_list + 1
  
    } else {
      
  
  dadesMeteo_sf <- st_transform(dadesMeteo_sf, crs = projeccio)
  
  dadesMeteo_sf_temp <- filter(dadesMeteo_sf, !is.na(max_temperature))
  dadesMeteo_sf_temp <- dadesMeteo_sf_temp %>% filter(!st_is_empty(.))
  
  dadesMeteo_sf_rh <- filter(dadesMeteo_sf, !is.na(mean_relative_humidity))
  dadesMeteo_sf_rh <- dadesMeteo_sf_rh %>% filter(!st_is_empty(.))
  
  dadesMeteo_sf_wind <- filter(dadesMeteo_sf, !is.na(mean_wind_speed))
  dadesMeteo_sf_wind <- dadesMeteo_sf_wind %>% filter(!st_is_empty(.))
  
  
  
  temp <- idw(max_temperature~1, dadesMeteo_sf_temp, grd)
  rh <- idw(mean_relative_humidity~1, dadesMeteo_sf_rh, grd)
  wind <- idw(mean_wind_speed~1, dadesMeteo_sf_wind, grd)
  
  temp <- as(temp, "Raster")
  rh <- as(rh, "Raster")
  wind <- as(wind, "Raster")
  
  
  temp_points <- extract(temp$var1.pred, grid_points_spatial_sf)
  rh_points <- extract(rh$var1.pred, grid_points_spatial_sf)
  wind_points <- extract(wind$var1.pred, grid_points_spatial_sf)
  
  # Incendis:
  fire_points = st_intersects(grid_points_spatial_sf, foc) # Comprova si interaccionen
  fire_intersects <- lengths(fire_points) > 0
  
  
  # Usos del sòl:
  usosSol_points <- extract(usosSol, grid_points_spatial_sf)
  
  # Geomorfologia:
  elevation_points <- extract(demCat, grid_points_spatial_sf)
  slopeCat_points <- extract(slopeCat, grid_points_spatial_sf)
  aspectCat_points <- extract(aspectCat, grid_points_spatial_sf)
  
  
  
  grid_df$Temperatura <- temp_points
  grid_df$Humitat <- rh_points
  grid_df$Vent <- wind_points
  grid_df$Incendi <- fire_intersects
  grid_df$UsosSol <- usosSol_points
  grid_df$Elevacio <- elevation_points
  grid_df$Pendent <- slopeCat_points
  grid_df$Orientació <- aspectCat_points
  
  
  grid_df$Date <- data
  
  
  df <- as.data.frame(grid_df)
  
  df_list[[n_list]] <- df
  
  n_list = n_list + 1
  
    }
  
}

IMPLEMENTACIÓ D’UNA XARXA NEURONAL

Després de valorar diferent llibreries i eines per a la implemantació de la Xarxa Neuronal s’ha obtat per utilitzar la llibreria H2O.

H2O és un producte creat per la companyia H2O.ai amb l’objectiu de combinar els principals algoritmes de machine learning i aprenentatge estadístic amb el Big Data. Gràcies a la seva forma de comprimir i emmagatzemar les dades, H2O és capaç de treballar amb milions de registres en un únic ordinador (utilitza tots els seus nuclis) o en un clúster de molts ordinadors. Internament, H2O està escrit en Java i segueix el paradigma Key/Value per emmagatzemar les dades i Map/Reduce pels seus algoritmes. Gràcies a les seves API, és possible accedir a totes les seves funcions des de R, Python o Scala, així com a través d’una interfície web anomenada Flow.

Tot i que l’avantatge principal de H2O davant d’altres eines és la seva escalabilitat, els seus algoritmes són igualment útils quan es treballa amb un volum de dades reduït.

Abans d’ajustar el model ha calgut adequar el dataset:

Disseny del model

En una Xarxa Neuronal hi ha molts paràmetres que es poden configurar. Alguns dels més importants són els següents:

  • Nombre de capes ocultes i nombre de nneurones per capes: S’ha provat amb combinacions de una o dues capes de 10, 20, 50, 80 i 100 neurones.

  • Estandarització: Les dades s’han estandaritzat per a què tinguin mitjana 0 i variancia 1 i així no hi hagi afectacions degudes a l’escala dels predictors.

  • Activació: Existeixen diverses funcions d’activació de les neurones de les capes intermitges (Tahn, Tahn with dropout, Rectifier, Rectifier with dropout, Maxout, Maxout with dropout). S’ha escollit la funció ‘Rectifier with dropout’, es tracta d’una funció simple i eficient, ja que activa només per valors positius, accelerant la convergència de l’entrenament.

  • Epochs: És el nombre d’iteracions d’aprenentatge durant l’entrenament de la xarxa. Trobar el valor òptim d’èpoques és molt important, ja que si són massa poques, la xarxa pot no aprendre prou, però si són massa nombroses, es produeix overfitting. Normalment, es pot tenir una idea aproximada del valor correcte en funció de la complexitat de la xarxa (com més complexa, més pesos cal aprendre) i el nombre d’observacions d’entrenament. Gràcies al sistema d’early stopping implementat en H2O, es pot indicar un nombre molt elevat d’èpoques sense risc d’overfitting, ja que l’entrenament s’aturarà quan la mètrica de validació triada no millori. Tot i així, sempre és recomanable representar l’evolució de l’error d’entrenament i validació en funció del nombre d’èpoques. En aquest cas hem triat 1000 iteracions.

# Hiperparàmetres a comparar
hiperparametres <- list(hidden = list(c(10), c(10,10), c(20), c(50), c(50,50), c(80), c(100,100)))
grid_dl <- h2o.grid(
              # Algoritme
              algorithm = "deeplearning",
              activation = "RectifierWithDropout",
              epochs = 500,
              # Variable resposta i predictors
              y = "Incendi",
              x = c("Elevacio", "Pendent", "Orientacio", "UsosSol", "Temperatura", "Vent", "Humitat"),
              # Dades d'entrenament.
              training_frame = dades_train_h2o,
              shuffle_training_data = FALSE,
              # Dades de validació
              validation_frame = dades_test_h2o,
              # Preprocessat
              standardize = TRUE,
              missing_values_handling = "Skip",
              # Detenció temprana
              stopping_rounds = 3,
              stopping_metric = "AUC",
              stopping_tolerance = 0.01,
              # Hiperparàmetres optimitzats
              hyper_params = hiperparametres,
              # Regularización
              l1 = 1e-5,
              l2 = 1e-5,
              # Tipus de cerca
              search_criteria = list(strategy = "Cartesian"),
              seed = 123,
              grid_id = "grid_dl"
          )
